# AGENTS.md — animus-sdk

## What this repo is
A Node.js/TypeScript SDK that implements the soul/mouth separation pattern for AI agents. The state engine runs locally and persistently; the LLM receives only a compiled mood-line paragraph.

## Genome (read before every session)
This repo uses the Cortex genome protocol. Read these files at session start — in order:

1. `.genome/00_GENOTYPE.md` — architecture invariants and hard lines (protected)
2. `.genome/10_PHENOTYPE.md` — current sprint focus, open questions, next actions (update freely)
3. `.genome/20_EPIGENOME.md` — append-only decision log (append only, never edit)
4. `.genome/30_SELECTION.md` — architect heuristics and autonomous action policy (protected)
5. `.genome/40_SHADOW.md` — rejected paths and forbidden zones (append only, never edit)

After reading, update `10_PHENOTYPE.md` with any changes to focus, open questions, or next actions before ending the session.

## Repo structure
- `bin/animus.js` — CLI entry point (`animus init`, `animus simulate`, `animus status`, `animus inject --from-cortex`)
- `schema/agent.schema.json` — JSON Schema for agent state definitions
- `src/` — SDK source (StateEngine, Compiler, Memory, EventSystem, Animus, adapters)
- `src/__tests__/` — test suite (run via `npm test`, compiles to `.dist-test/`)
- `dist/` — compiled CommonJS output (generated by `npm run build`)
- `simulator/` — self-contained Canvas browser visualizer (no build step, runs as file://)
- `templates/` — files copied into user projects by `animus init`
- `examples/quickstart/` — minimal working TypeScript demo

## Build and test
```bash
npm run build   # tsc → dist/ (no __tests__ compiled here)
npm test        # tsc -p tsconfig.test.json → .dist-test/, then node --test
```

## Core invariants (do not change without architect sign-off)
1. **The mood-line is the only interface between state engine and LLM.** The LLM never receives raw state variable values.
2. **State updates are driven by events, not by LLM output text.** The response is parsed for event tags; raw text never modifies state directly.
3. **The state engine runs offline.** No state update requires a network call.
4. **Memory file is `agent.memory.json` — plain JSON, atomic write.** Never `.db`, never binary.
5. **Zero runtime dependencies.** No third-party packages in `src/` or `bin/`.
